ZITADEL Console already provides a built-in "Delete Account" button on the user's profile page for self-service account deletion. This guide focuses on implementing user self-deletion functionality in your custom applications using ZITADEL's APIs.

## Implementation Approach

### User v2 API-Based Deletion

You can build a custom interface in your application that allows authenticated users to delete their own accounts using ZITADEL's User v2 API.

**Requirements:**
- User authentication (the user must be logged in)
- User identity verification (e.g., password confirmation or MFA)
- API call to delete the user

**Steps:**

1. Authenticate the user in your application
2. Verify the user's identity (e.g., require password confirmation or MFA)
3. Call the [User v2 DeleteUser endpoint](https://zitadel.com/docs/reference/api/user) (`DELETE /v2/users/{user_id}`) with the user's own access token
4. Handle the session termination and redirect

**Key Points:**
- An authenticated user can delete their own account without requiring special administrative roles
- The user's access token provides the necessary authorization
- No service account is needed for self-deletion

### Enhancing with Actions for Compliance

You can use [ZITADEL Actions](https://zitadel.com/docs/guides/manage/console/actions-overview) as a compliance and audit add-on to log deletion requests or trigger additional workflows alongside the API-based deletion.

**Example Action for Audit Logging:**

```js
/**
 * Log account deletion for compliance
 * 
 * Flow: Custom trigger based on your implementation
 * 
 * @param ctx
 * @param api
 */
function logAccountDeletion(ctx, api) {
  // Log the deletion request for audit purposes
  var logger = require("zitadel/log");
  logger.log("Account deletion requested for user: " + ctx.v1.user.id);
  
  // Optionally notify external compliance systems
  var http = require('zitadel/http');
  http.fetch('YOUR_AUDIT_API_ENDPOINT', {
    method: 'POST',
    body: {
      userId: ctx.v1.user.id,
      userName: ctx.v1.user.userName,
      timestamp: new Date().toISOString()
    }
  });
}
```

**Note:** Actions complement the API-based deletion by providing audit trails and triggering compliance workflows. The actual deletion is performed via the User v2 API call.

## Important Considerations

### Data Retention and Compliance

- Ensure compliance with data protection regulations (GDPR, CCPA, etc.)
- Consider implementing a grace period before permanent deletion
- Maintain audit logs of deletion requests using ZITADEL's [audit trail](https://zitadel.com/docs/guides/integrate/external-audit-log)

### Session Management

After account deletion:
- Terminate all active sessions
- Revoke all tokens
- Redirect the user appropriately

### Verification Steps

Before deletion, consider requiring:
- Password re-authentication
- MFA verification
- Explicit confirmation (e.g., typing "DELETE" or account email)

## API Reference

To delete a user programmatically, use the User v2 API:

- **Endpoint:** `DELETE /v2/users/{user_id}`
- **Authorization:** User's own access token (no special role required for self-deletion)
- **Documentation:** [User v2 API](https://zitadel.com/docs/reference/api/user)
