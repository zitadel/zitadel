This sample app implements the [OIDC PKCE](/docs/guides/integrate/login/oidc/login-users) flow for secure authentication.
- Login: Exchanges a code challenge for an access token via ZITADEL.
- Authorize: Uses the token to retrieve profile data from the userinfo endpoint.
- Logout: Clears the local session and revokes the token.

## Create your React application with ZITADEL OIDC PKCE authentication

### The Authentication Flow
Here is how your React app interacts with ZITADEL using OIDC and PKCE:
1. **Login**: The user clicks "Login," and the app redirects them to ZITADEL with a PKCE challenge.
2. **Auth**: The user logs in via the ZITADEL hosted page.
3. **Code Exchange**: ZITADEL redirects back with an auth code. The app exchanges this code (and the PKCE verifier) for an access token.
4. **Profile**: The app calls the /userinfo endpoint with the token to get the user's name and email.
5. **Logout**: The user logs out; the app clears the token and ends the ZITADEL session.

### 1. Prerequisites

Ensure you have the following installed on your machine:
- [Node.js]((https://nodejs.org/)): Version 16 or later is recommended.
- [Visual Studio Code]((https://code.visualstudio.com/)): Or your preferred code editor.

### 2. Get the Example App

To get you up and running quickly, we‚Äôve prepared a React boilerplate using the Zitadel react sdk.
```bash
git clone https://github.com/zitadel/zitadel-react.git
cd zitadel-react
```

### 3. Build the SDK

Navigate to the `/lib` folder and run `yarn install` and `yarn build`.

```bash
cd ./lib
yarn install
yarn build
```

Then, navigate back and install the dependencies

```bash
cd ..
yarn install
```

### 4. Add client ID and issuer

Open `src/App.tsx` and replace the placeholder for *authority* and *client_id* with the values from the previous steps (issuer, client id):

```js
const config: ZitadelConfig = {
    authority: "https://CUSTOM_DOMAIN",
    client_id: "YOUR_CLIENT_ID",
    redirect_uri: "http://localhost:3000/callback",
    post_logout_redirect_uri: "http://localhost:3000",
    response_type: 'code',
    scope: 'openid profile email'
};
```

### 5. Start the app

```bash
yarn start
```

Your application will then run on `http://localhost:3000`
