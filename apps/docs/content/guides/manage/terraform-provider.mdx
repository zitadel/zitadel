The [Zitadel Terraform Provider](https://registry.terraform.io/providers/zitadel/zitadel/latest/docs) lets you manage Zitadel resources — organizations, projects, applications, users, identity providers, policies, and more — using Terraform's declarative configuration language.

The provider communicates with the [ZITADEL API](/apis/introduction), so any resource you can configure through the Console or API can also be managed as code.

## Before you start

- [Install Terraform](https://developer.hashicorp.com/terraform/install) (v1.0+)
- Have a running [Zitadel instance](/guides/start/quickstart)
- Create a [service account](/guides/integrate/service-accounts/authenticate-service-accounts) with [sufficient authorization](/guides/manage/console/administrators) to manage the desired resources

## Configure the provider

Create a `main.tf` file and configure the provider block:

```hcl
terraform {
  required_providers {
    zitadel = {
      source  = "zitadel/zitadel"
      version = "~> 2.0"
    }
  }
}

provider "zitadel" {
  domain           = "my-instance.zitadel.cloud"
  jwt_profile_file = "service-user.json"
}
```

### Authentication methods

The provider supports several authentication options. Use exactly one:

| Method | Configuration | Description |
| --- | --- | --- |
| JWT Profile File | `jwt_profile_file = "path/to/key.json"` | Path to a downloaded service-account key file. Recommended for production. |
| JWT Profile JSON | `jwt_profile_json = "{...}"` | Inline JSON string of the service-account key. Useful for CI/CD secrets. |
| Presigned JWT | `jwt_file = "path/to/jwt"` | Path to a file containing a presigned JWT. |
| Personal Access Token | `access_token = "PAT_VALUE"` | A personal access token. Suitable for development and testing. |

### Provider options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `domain` | string | — | Domain of your Zitadel instance (required). |
| `port` | string | — | Used port if not the default ports 80 or 443 are configured. |
| `insecure` | bool | `false` | Use insecure (HTTP) connection. |
| `insecure_skip_verify_tls` | bool | `false` | Disable TLS certificate verification. Only use in development or testing. |
| `access_token` | string | — | Personal Access Token to connect to Zitadel. |
| `jwt_profile_file` | string | — | Path to a service-account key JSON file. |
| `jwt_profile_json` | string | — | Service-account key JSON as an inline string. |
| `jwt_file` | string | — | Path to a file containing a presigned JWT. |
| `token` | string | — | Deprecated. Path to a JWT profile file. Use `jwt_profile_file` instead. |
| `transport_headers` | map | — | Custom headers to add to HTTP and gRPC requests. |

## Create resources

After configuring the provider, declare Zitadel resources in your Terraform files. The example below creates an organization and a human user:

```hcl
resource "zitadel_org" "demo" {
  name = "Demo Organization"
}

resource "zitadel_human_user" "alice" {
  org_id             = zitadel_org.demo.id
  user_name          = "alice@example.com"
  first_name         = "Alice"
  last_name          = "Example"
  display_name       = "Alice Example"
  preferred_language = "en"
  email              = "alice@example.com"
  is_email_verified  = true
}
```

## Apply the configuration

Run Terraform in the directory containing your `.tf` files:

```bash
# Initialize the provider
terraform init

# Preview the changes
terraform plan

# Apply the changes
terraform apply
```

To remove all resources managed by Terraform:

```bash
terraform destroy
```

## Available resources

The provider covers a wide range of Zitadel resources including:

- **Organizations** — `zitadel_org`
- **Users** — `zitadel_human_user`, `zitadel_machine_user`
- **Projects and applications** — `zitadel_project`, `zitadel_application_oidc`, `zitadel_application_api`, `zitadel_application_saml`
- **Identity providers** — `zitadel_idp_google`, `zitadel_idp_azure_ad`, `zitadel_idp_github`, `zitadel_idp_gitlab`, `zitadel_idp_ldap`, `zitadel_idp_saml`, and more
- **Policies** — `zitadel_login_policy`, `zitadel_password_complexity_policy`, `zitadel_label_policy`, `zitadel_privacy_policy`, and more
- **Actions** — `zitadel_action`, `zitadel_trigger_actions`
- **Access control** — `zitadel_user_grant`, `zitadel_org_member`, `zitadel_project_role`

Browse the full list of resources and data sources on the [Terraform Registry](https://registry.terraform.io/providers/zitadel/zitadel/latest/docs).
