## Overview

[React](https://react.dev) SPAs can use OIDC Authorization Code + PKCE with ZITADEL. The core challenge is runtime boundaries: a SPA can render auth UI in-browser, but sensitive operations should run on a server/BFF.

This repository includes a React Router + Vite example that uses a shared BFF:

- SPA UI: `examples/react-router`
- Shared server/BFF: `examples/spa-bff`

## Browser vs server responsibilities

| Responsibility | Runtime |
| --- | --- |
| Sign-in button UX, route-level auth state, profile rendering | Browser (React SPA) |
| OIDC callback endpoint and code exchange side effects | Server/BFF |
| Session cookie creation/invalidations | Server/BFF |
| Confidential credentials (service user, private keys) | Server/BFF |
| Webhook verification and management API calls | Server/BFF |

This split keeps secrets and high-trust operations out of browser code.

## Run the in-repo React example

1. Configure the BFF env:

```sh
cp examples/spa-bff/.env.example examples/spa-bff/.env.local
```

2. Set these values in `examples/spa-bff/.env.local`:

```env
SPA_ORIGIN=http://localhost:5173
ZITADEL_ISSUER_URL=https://your-instance.zitadel.cloud
ZITADEL_CLIENT_ID=your-client-id
ZITADEL_CALLBACK_URL=http://localhost:3001/auth/callback
```

3. Start the BFF and SPA:

```sh
pnpm --filter @zitadel/examples/spa-bff dev
pnpm --filter @zitadel/examples/react-router dev
```

Open `http://localhost:5173`.
