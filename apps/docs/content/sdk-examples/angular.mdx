## Overview

[Angular](https://angular.dev) SPAs can integrate with ZITADEL using OIDC + PKCE and a minimal BFF for server-required operations.

This repository includes an Angular example that shares the same BFF contract as the React Router example:

- Angular SPA: `examples/angular`
- Shared server/BFF: `examples/spa-bff`

## Browser vs server responsibilities

| Responsibility | Runtime |
| --- | --- |
| Route guards and auth UI interactions | Browser (Angular app) |
| Authorization callback handling and code exchange side effects | Server/BFF |
| Session cookie creation and lifecycle | Server/BFF |
| Token introspection/service credentials/private keys | Server/BFF |
| Webhook verification and admin API writes | Server/BFF |

## Run the in-repo Angular example

1. Configure the BFF env:

```sh
cp examples/spa-bff/.env.example examples/spa-bff/.env.local
```

2. Set these values in `examples/spa-bff/.env.local`:

```env
SPA_ORIGIN=http://localhost:4200
ZITADEL_ISSUER_URL=https://your-instance.zitadel.cloud
ZITADEL_CLIENT_ID=your-client-id
ZITADEL_CALLBACK_URL=http://localhost:3001/auth/callback
```

3. Start the BFF and Angular app:

```sh
pnpm --filter @zitadel/examples/spa-bff dev
pnpm --filter @zitadel/examples/angular dev
```

Open `http://localhost:4200`.

## Notes on SDK layering

- `@zitadel/angular` is the Angular-facing SDK surface.
- `@zitadel/zitadel-js` remains the framework-agnostic core.
- In SPA architectures, sensitive operations still belong in server routes/BFF endpoints even when framework adapters are available.
