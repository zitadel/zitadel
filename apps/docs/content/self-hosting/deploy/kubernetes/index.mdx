This guide takes you from zero to a running ZITADEL instance on Kubernetes and then shows you how to harden it for production.

## Stage 1 — Local Quickstart (~5 minutes)

This stage uses [k3d](https://k3d.io/) — a lightweight Kubernetes distribution (k3s) that runs in Docker and bundles Traefik as its ingress controller. The ZITADEL chart ships with an optional in-cluster PostgreSQL, so a single `helm install` deploys the full stack.

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) Engine 24+
- [k3d](https://k3d.io/#installation) 5+
- [kubectl](https://kubernetes.io/docs/tasks/tools/)
- [Helm](https://helm.sh/docs/intro/install/) 3.x or 4.x

### Create a cluster

The `--port` flag maps port 8080 on your machine to the cluster's HTTP load balancer:

```bash
k3d cluster create zitadel --port "8080:80@loadbalancer"
```

### Install the full stack

```bash
helm repo add zitadel https://charts.zitadel.com
helm repo add bitnami https://charts.bitnami.com/bitnami
helm repo update

helm install zitadel zitadel/zitadel \
  --set postgresql.enabled=true \
  --set zitadel.masterkey="MasterkeyNeedsToHave32Characters" \
  --set zitadel.configmapConfig.ExternalDomain=localhost \
  --set zitadel.configmapConfig.ExternalPort=8080 \
  --set zitadel.configmapConfig.ExternalSecure=false \
  --set zitadel.configmapConfig.TLS.Enabled=false \
  --set ingress.enabled=true \
  --set ingress.className=traefik \
  --set login.ingress.enabled=true \
  --set login.ingress.className=traefik \
  --wait
```

Or save the following as `quickstart-values.yaml` and run `helm install zitadel zitadel/zitadel --values quickstart-values.yaml --wait`:

```yaml
postgresql:
  enabled: true
  auth:
    database: zitadel
    username: zitadel
    password: zitadel
    postgresPassword: zitadel
  primary:
    persistence:
      enabled: false

zitadel:
  masterkey: "MasterkeyNeedsToHave32Characters"
  configmapConfig:
    ExternalDomain: localhost
    ExternalPort: 8080
    ExternalSecure: false
    TLS:
      Enabled: false
    FirstInstance:
      Org:
        Human:
          UserName: zitadel-admin
          Email: zitadel-admin@zitadel.localhost
          Password: "Password1!"
          PasswordChangeRequired: false

ingress:
  enabled: true
  className: traefik
  hosts:
    - host: localhost
      paths:
        - path: /
          pathType: Prefix

login:
  ingress:
    enabled: true
    className: traefik
    hosts:
      - host: localhost
        paths:
          - path: /ui/v2/login
            pathType: Prefix
```

That's it. Visit [http://localhost:8080/ui/console](http://localhost:8080/ui/console) to open the console.

<Callout type="warn">
The masterkey [encrypts sensitive data at rest](/concepts/architecture/secrets). The value above is a **dev placeholder** — generate a real one before any non-local deployment:
```bash
tr -dc A-Za-z0-9 </dev/urandom | head -c 32
```
Once ZITADEL has been initialized with a masterkey, it **cannot be changed** without losing access to encrypted data.
</Callout>

<Callout type="info">
The stack runs: **Traefik** ([reverse proxy](/self-hosting/manage/reverseproxy/reverse_proxy)) → **ZITADEL API** (Go) + **ZITADEL Login** (Next.js) → **PostgreSQL**.
[gRPC over HTTP/2](/self-hosting/manage/http2) is handled automatically by Traefik — no extra configuration needed.
</Callout>

### Swap out components

The bundled PostgreSQL (`postgresql.enabled=true`) is for local use only. Once you're ready to move forward, you can replace individual components:

| Component | How to replace |
|-----------|---------------|
| **Database** | Set `postgresql.enabled=false` and configure `zitadel.configmapConfig.Database` pointing to your own PostgreSQL |
| **Reverse proxy** | Set `ingress.className` to `nginx`, `alb`, or your controller of choice |
| **Caching** | Add Redis by configuring `zitadel.configmapConfig.Caches` (see [Caching](/self-hosting/deploy/kubernetes/caching)) |

## Stage 2 — Production Cluster

For production you need:

- A Kubernetes cluster (1.30+)
- An ingress controller (Traefik, NGINX, or cloud-native)
- A domain with DNS configured
- TLS certificates (via cert-manager, ACME, or manually)
- PostgreSQL 14+ (managed service or in-cluster)
- Helm 3.x+

<Callout type="warn">
The ZITADEL management console [requires end-to-end HTTP/2 support](/self-hosting/manage/http2). Ensure your ingress controller is configured to forward HTTP/2 (h2c) traffic to ZITADEL pods.
</Callout>

### Create secrets

```bash
# Masterkey — generate once, store safely, cannot be changed after first run
kubectl create secret generic zitadel-masterkey \
  --from-literal=masterkey="$(tr -dc A-Za-z0-9 </dev/urandom | head -c 32)"

# Database credentials
kubectl create secret generic zitadel-db-credentials \
  --from-literal=config.yaml="
Database:
  Postgres:
    User:
      Password: your-secure-app-password
    Admin:
      Password: your-secure-admin-password
"
```

### Configure values

Save the following as `values.yaml`. Replace `zitadel.example.com` with your domain:

```yaml
replicaCount: 2

zitadel:
  masterkeySecretName: zitadel-masterkey
  configSecretName: zitadel-db-credentials
  configmapConfig:
    ExternalDomain: "zitadel.example.com"
    ExternalSecure: true
    ExternalPort: 443
    TLS:
      Enabled: false
    Database:
      Postgres:
        Host: "postgres.database.svc.cluster.local"
        Port: 5432
        Database: "zitadel"
        User:
          Username: "zitadel"
          SSL:
            Mode: "verify-full"
        Admin:
          Username: "postgres"
          SSL:
            Mode: "verify-full"
    FirstInstance:
      Org:
        Human:
          UserName: "admin"
          Email: "admin@example.com"
          Password: "YourSecurePassword123!"
          PasswordChangeRequired: true

ingress:
  enabled: true
  className: traefik
  annotations:
    traefik.ingress.kubernetes.io/router.entrypoints: websecure
    traefik.ingress.kubernetes.io/router.tls: "true"
    traefik.ingress.kubernetes.io/router.tls.certresolver: letsencrypt
  hosts:
    - host: zitadel.example.com
      paths:
        - path: /
          pathType: Prefix
  tls:
    - secretName: zitadel-tls
      hosts:
        - zitadel.example.com

login:
  ingress:
    enabled: true
    className: traefik
    annotations:
      traefik.ingress.kubernetes.io/router.entrypoints: websecure
      traefik.ingress.kubernetes.io/router.tls: "true"
      traefik.ingress.kubernetes.io/router.tls.certresolver: letsencrypt
    hosts:
      - host: zitadel.example.com
        paths:
          - path: /ui/v2/login
            pathType: Prefix
    tls:
      - secretName: zitadel-tls
        hosts:
          - zitadel.example.com

podDisruptionBudget:
  enabled: true
  minAvailable: 1
```

### Install

```bash
helm repo add zitadel https://charts.zitadel.com && helm repo update

helm install zitadel zitadel/zitadel --values values.yaml --wait
```

### Verify

Watch the pods come up:

```bash
kubectl get pods --watch
```

You should see the `zitadel-init` and `zitadel-setup` jobs complete, followed by the `zitadel` deployment pods becoming Ready.

Check the Helm release status:

```bash
helm status zitadel
```

Access the console at `https://zitadel.example.com/ui/console`.
